# СЛАЙД 5: ТЕХНОЛОГИИ
**Тайминг: 20 секунд**

## Технологический стек

| Слой | Технология | Зачем |
|------|-----------|-------|
| Frontend | React + TypeScript + Tailwind CSS v4 | Быстрый, отзывчивый интерфейс |
| Backend | Django 5 + DRF | Надежный API, быстрая разработка |
| LLM | Claude API (claude-sonnet-4) | Лучшее понимание русского юридического текста |
| База данных | PostgreSQL (Alem Plus) | Хранение тендеров и анализов |
| Кеш | Redis (Alem Plus) | Мгновенный повторный анализ (MD5-хеш PDF) |
| Эмбеддинги | Alem Plus Embedder API | 1024-мерные векторы для поиска |
| Векторный поиск | Milvus (Alem Plus) | Поиск похожих тендеров для RAG |
| CI/CD | GitLab (Alem Plus) | Деплой на gitlabapp.alem.ai |

## Архитектура AI Pipeline

```
PDF документ
    │
    ▼
PyMuPDF (извлечение текста)
    │
    ├──────────────────────────────────┐
    ▼                                  ▼
Claude AI                     Embedder API (Alem Plus)
  ├─ Извлечение требований           │
  ├─ Оценка рисков (0-100)           ▼
  ├─ Поиск подводных камней     Milvus (Alem Plus)
  └─ Генерация предложения       (векторное хранилище)
    ▲                                  │
    │                                  │
    └──── RAG-контекст ◄──────────────┘
          (похожие тендеры)

    Кеш: Redis хранит ответы Claude по MD5
    Повторный анализ = 0 секунд
```

## Процесс разработки
- **48 часов** хакатона
- **5 инструментов** Alem Plus
- **50+** реальных тендеров протестировано
- CI/CD через GitLab Alem Plus с автодеплоем

**Заметки спикера:**
- Не углубляться в технические детали — жюри ценит результат
- Ключевой посыл: "Мы используем 5 инструментов Alem Plus — максимум из возможного"
- Redis-кеш — конкретное преимущество: "Повторный анализ того же PDF — мгновенный"

# Q&A — Возможные вопросы жюри (2 минуты)

**Q: Почему Claude, а не GPT?**

**Q: Как вы зарабатываете?**
